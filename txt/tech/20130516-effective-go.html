Effective GO 阅读要点

godoc -http=:8080

格式

  • 没有必要纠结于格式，gofmt 可以帮助我们做这些烦人的事情
  • 不用纠结于一行的长度
  • 不用像c一样用一堆括号
  • 自动行末分号

注释

  • 注释即是文档，不用纠结于格式
  • 将相关变量集中到{}内

命名

  • package 命名一般要短而精，小写字母，单word
  • package 名即是文件名
  • 注释文档比长命名有效
  • 大写字母开头的函数为 package 对外接口
  • getter 函数不用 get，可以 set
  • interface 函数常以 er 结尾
  • mixCap or MixCap > mix_cap

语句

  • 没有 while 或者 do...while，使用 for 实现
  • := 至少有一个是新定义的变量
  • for key(_), value range map:
  • swith 的用法被大大扩展，无需 break，可作为多重判断，无所谓是 int 类型
  • defer 的执行顺序是LIFO

数据

  • make 用于 map, slice, channel 的内存分配 “需要初始化(长度等信息)”
  • new 分配内存，zero，返回第一个地址指针
  • 函数传递 array 时会进行拷贝操作，而不是指针
  • slice > array
  • map 是无序的
  • struct 的匿名字段可以用于继承

Print

  • %v 可以作为通用类型输出
  • %T 变量的类型

Interface

  • 有助于实现 ducking type
  • type switch value.(typename)
  • checking val.(interface)

go routine

  • channel 缓冲或者不缓冲
  • channel 可以通过 range 遍历，直到生产者 close
  • share by communication
